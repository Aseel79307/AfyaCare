#  ملخص تحديثات قسم الـ Backend (Medical Chatbot)

هذا الملف يحتوي على ملخص لجميع التحديثات الجوهرية التي تمت على نظام الشات بوت الطبي، بالإضافة إلى شرح لهيكلة الملفات الجديدة وطريقة تشغيل المشروع لتسهيل عملية الربط القادمة مع قاعدة البيانات **Supabase**.

---

## 📂 الهيكلة الجديدة للمشروع
تمت إعادة بناء المشروع بالكامل ليصبح Modular (مقسم ومرتب) لتسهيل العمل عليه وإضافة الميزات مستقبلاً. الهيكلة الحالية هي:

```text
medical-chatbot/
├── data/                  # الملفات الخام (مثل Medical_book.pdf) وبعض المخرجات
├── database/              # ملفات قاعدة بيانات المتجهات المحلية (FAISS: .index و .pkl)
├── core/                  # الأدوات الأساسية للنظام
│   ├── embeddings.py      # مسؤول عن تحويل النصوص إلى متجهات (Embeddings)
│   ├── pdf_processor.py   # مسؤول عن قراءة وتقسيم الـ PDF
│   └── state.py           # يحتوي على المتغيرات العامة (Global State) مثل embedding_manager
├── rag/                   # منطق الـ RAG والـ API
│   ├── models.py          # فئات Pydantic (مثل ChatRequest, ChatResponse...)
│   ├── services.py        # دوال المساعدة للتعامل مع الذكاء الاصطناعي (Prompts & Parsing)
│   └── routes.py          # أزرار النهاية (Endpoints) الخاصة بالـ API باستثناء المحادثة
├── agent/                 # منطق الـ Agentic RAG (الذكاء الاصطناعي المستقل)
│   ├── tools.py           # الأدوات التي يمتلكها الذكاء الاصطناعي (مثل دالة البحث search_medicaldb)
│   └── core.py            # الحلقة الأساسية (ReAct Loop) للـ Agent
├── main.py                # نقطة الدخول الرئيسية، صغيرة ومخصصة لتشغيل السيرفر فقط
├── requirements.txt
└── .env                   # يجب أن يحتوي على OPENROUTER_API_KEY
```

---

##  التحول إلى Agentic RAG
في السابق، كان السيرفر يقوم بالبحث في الـ PDF في كل مرة ويقوم بعمل (Naive RAG). 
الآن تم تحويل النظام إلى **Agentic RAG** متطور يعتمد على **Function Calling**:
1. يستقبل السيرفر رسالة المريض بالإضافة لسجل المحادثة (`chat_history`) ليحفظ السياق.
2. يستخدم النماذج عبر OpenRouter (تم الاعتماد على `openai/gpt-oss-120b:free`).
3. يقوم الـ Agent بالتفكير التلقائي: هل أحتاج للبحث في قاعدة البيانات الطبية؟ 
4. إذا نعم، يقوم الـ Agent تلقائياً باستدعاء الدالة `search_medicaldb`، يقرأ النتيجة، ثم يرد على المريض.
5. الـ Agent مبرمج الآن ليقوم بمحادثة تدريجية (Step-by-Step) وليسرد إجابته وينهيها دائماً بـ **سؤال توضيحي واحد** للمريض حتى يتوصل للتشخيص المبدئي الدقيق.

##  كيف تبدأ وتشغل المشروع محلياً؟

لتشغيل السيرفر بشكل صحيح وتفعيل البيئة الوهمية (Virtual Environment) في الـ PowerShell الخاصة بالـ Windows، تأكد دائماً أنك تقف داخل مجلد `medical-chatbot`.

### 1. الوصول للمسار الصحيح والتفعيل:
```powershell
# الوصول للمجلد
cd C:\Users\abuha\Desktop\Afya_Care\backend\medical-chatbot

# تفعيل البيئة الوهمية
.\venv\Scripts\Activate.ps1
```
*(ملاحظة: إذا ظهر لك خطأ في التفعيل، تأكد أنك داخل مجلد `medical-chatbot` وليس `backend` العادي)*

### 2. تشغيل السيرفر:
بعد ظهور كلمة `(venv)` في بداية السطر في الـ PowerShell، يمكنك تشغيل السيرفر بهذا الأمر:
```powershell
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```
وسيعمل السيرفر على الرابط: `http://localhost:8000`

## 🔗 الخطوة القادمة للفريق (Supabase)
الملفات الآن جاهزة تماماً للعمل المستقبلي.
يمكنكم الآن استكمال العمل في مجلدات الـ `database/` أو إضافة أدوات جديدة للـ Agent في ملف `agent/tools.py` لكي يقوم بالبحث في أو حفظ بيانات المريض مباشرة داخل **Supabase**. بالتوفيق! 🚀
